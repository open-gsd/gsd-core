# 續接格式

完成命令或工作流後展示下一步的標準格式。

## 核心結構

```
---

## ▶ 下一步

**{識別符號}: {名稱}** — {單行描述}

`{可複製貼上的命令}`

<sub>`/clear` 優先 → 全新上下文視窗</sub>

---

**也可選：**
- `{備選項 1}` — 描述
- `{備選項 2}` — 描述

---
```

## 格式規則

1. **始終展示它是什麼** — 名稱 + 描述，絕不僅僅是一個命令路徑
2. **從原始檔拉取上下文** — ROADMAP.md 用於階段，PLAN.md `<objective>` 用於計劃
3. **命令用內聯程式碼** — 反引號，易於複製貼上，渲染為可點選連結
4. **`/clear` 說明** — 始終包含，保持簡潔但解釋原因
5. **用"也可選"而非"其他選項"** — 聽起來更像應用
6. **視覺分隔符** — 上下用 `---` 使其突出

## 變體

### 執行下一個計劃

```
---

## ▶ 下一步

**02-03: 重新整理令牌輪換** — 新增帶滑動過期的 /api/auth/refresh

`/gsd-execute-phase 2`

<sub>`/clear` 優先 → 全新上下文視窗</sub>

---

**也可選：**
- 執行前審查計劃
- `/gsd-discuss-phase --assumptions 2` — 檢查假設

---
```

### 執行階段中最後一個計劃

添加註釋說明這是最後一個計劃以及接下來是什麼：

```
---

## ▶ 下一步

**02-03: 重新整理令牌輪換** — 新增帶滑動過期的 /api/auth/refresh
<sub>階段 2 的最後一個計劃</sub>

`/gsd-execute-phase 2`

<sub>`/clear` 優先 → 全新上下文視窗</sub>

---

**完成後：**
- 階段 2 → 階段 3 過渡
- 下一步：**階段 3: 核心功能** — 使用者儀表板和設定

---
```

### 規劃階段

```
---

## ▶ 下一步

**階段 2: 認證** — 帶重新整理令牌的 JWT 登入流程

`/gsd-plan-phase 2`

<sub>`/clear` 優先 → 全新上下文視窗</sub>

---

**也可選：**
- `/gsd-discuss-phase 2` — 先收集上下文
- `/gsd-plan-phase --research-phase 2` — 調查未知項
- 審查路線圖

---
```

### 階段完成，準備下一步

在下一步操作前顯示完成狀態：

```
---

## ✓ 階段 2 完成

3/3 計劃已執行

## ▶ 下一步

**階段 3: 核心功能** — 使用者儀表板、設定和資料匯出

`/gsd-plan-phase 3`

<sub>`/clear` 優先 → 全新上下文視窗</sub>

---

**也可選：**
- `/gsd-discuss-phase 3` — 先收集上下文
- `/gsd-plan-phase --research-phase 3` — 調查未知項
- 回顧階段 2 構建的內容

---
```

### 多個同等選項

當沒有明確的主要操作時：

```
---

## ▶ 下一步

**階段 3: 核心功能** — 使用者儀表板、設定和資料匯出

**直接規劃：** `/gsd-plan-phase 3`

**先討論上下文：** `/gsd-discuss-phase 3`

**研究未知項：** `/gsd-plan-phase --research-phase 3`

<sub>`/clear` 優先 → 全新上下文視窗</sub>

---
```

### 里程碑完成

```
---

## 🎉 里程碑 v1.0 完成

全部 4 個階段已釋出

## ▶ 下一步

**開始 v1.1** — 提問 → 研究 → 需求 → 路線圖

`/gsd-new-milestone`

<sub>`/clear` 優先 → 全新上下文視窗</sub>

---
```

## 拉取上下文

### 用於階段（從 ROADMAP.md）：

```markdown
### 階段 2: 認證
**目標**: 帶重新整理令牌的 JWT 登入流程
```

提取：`**階段 2: 認證** — 帶重新整理令牌的 JWT 登入流程`

### 用於計劃（從 ROADMAP.md）：

```markdown
計劃:
- [ ] 02-03: 新增重新整理令牌輪換
```

或從 PLAN.md `<objective>`：

```xml
<objective>
新增帶滑動過期視窗的重新整理令牌輪換。

目的: 在不影響安全性的前提下延長會話生命週期。
</objective>
```

提取：`**02-03: 重新整理令牌輪換** — 新增帶滑動過期的 /api/auth/refresh`

## 反模式

### 不要：僅命令（無上下文）

```
## 繼續

執行 `/clear`，然後貼上：
/gsd-execute-phase 2
```

使用者不知道 02-03 是關於什麼的。

### 不要：缺少 /clear 說明

```
`/gsd-plan-phase 3`

先執行 /clear。
```

沒有解釋原因。使用者可能跳過。

### 不要："其他選項" 措辭

```
其他選項：
- 審查路線圖
```

聽起來像是事後補充。用"也可選："替代。

### 不要：用圍欄程式碼塊展示命令

```
```
/gsd-plan-phase 3
```
```

模板內的圍欄程式碼塊會造成巢狀歧義。用內聯反引號替代。
