# UI 品牌規範

面向使用者的 GSD 輸出的視覺模式。編排器通過 @ 引用此檔案。

## 階段橫幅

用於主要工作流過渡。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► {階段名稱}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**階段名稱（大寫）：**
- `QUESTIONING`（提問）
- `RESEARCHING`（研究）
- `DEFINING REQUIREMENTS`（定義需求）
- `CREATING ROADMAP`（建立路線圖）
- `PLANNING PHASE {N}`（規劃階段 {N}）
- `EXECUTING WAVE {N}`（執行波次 {N}）
- `VERIFYING`（驗證）
- `PHASE {N} COMPLETE ✓`（階段 {N} 完成）
- `MILESTONE COMPLETE 🎉`（里程碑完成）

---

## 檢查點框

需要使用者操作。62 字元寬度。

```
╔══════════════════════════════════════════════════════════════╗
║  CHECKPOINT: {型別}                                          ║
╚══════════════════════════════════════════════════════════════╝

{內容}

──────────────────────────────────────────────────────────────
→ {操作提示}
──────────────────────────────────────────────────────────────
```

**型別：**
- `CHECKPOINT: 需要驗證` → `→ 輸入 "approved" 或描述問題`
- `CHECKPOINT: 需要決策` → `→ 選擇: option-a / option-b`
- `CHECKPOINT: 需要操作` → `→ 完成後輸入 "done"`

---

## 狀態符號

```
✓  完成 / 通過 / 已驗證
✗  失敗 / 缺失 / 阻塞
◆  進行中
○  待處理
⚡ 自動批准
⚠  警告
🎉 里程碑完成（僅在橫幅中）
```

- (根據工作流選填可選命令，例如 `/gsd-progress --next`)

## 反模式

- 變化的框/橫幅寬度
- 混合橫幅樣式（`===`、`---`、`***`）
- 橫幅中缺少 `GSD ►` 字首
- 隨機 emoji（`🚀`、`✨`、`💫`）
- 完成後缺少下一步區塊
