# 如何自主執行階段

無需人工干預地執行所有剩餘階段——或指定範圍內的階段——GSD 將自動完成每個階段的討論 → 規劃 → 執行流程。

有關自主執行期間階段迴圈的工作原理，請參閱[階段迴圈](../explanation/the-phase-loop.md)。

---

## 前提條件

- 已有包含 `.planning/ROADMAP.md` 和 `.planning/STATE.md` 的活躍專案
- 所有要執行的階段必須處於自主模式可驅動的狀態（待處理或進行中；非已完成狀態）
- 您關心的所有設計決策應已記錄在 `PROJECT.md` 中，或通過之前的 `/gsd-discuss-phase` 捕獲——自主模式僅在使用 `--interactive` 時才能互動式地處理灰色地帶

---

## 執行所有剩餘階段

```bash
/gsd-autonomous
```

GSD 讀取 `ROADMAP.md`，按數字順序發現所有未完成的階段，並對每個階段執行討論 → 規劃 → 執行。所有階段完成後，它會自動執行里程碑生命週期：審計 → 完成 → 清理。

---

## 執行特定範圍的階段

使用 `--from` 和 `--to` 來限定執行範圍。兩個標誌都接受十進位制階段編號（例如 `3.1`）。

```bash
/gsd-autonomous --from 3          # 階段 3、4、5 …（跳過已完成的階段 1 和 2）
/gsd-autonomous --to 5            # 直到幷包括階段 5
/gsd-autonomous --from 3 --to 5   # 恰好是階段 3、4 和 5
```

到達 `--to` 時，生命週期步驟會被跳過，因為並非所有里程碑階段都已完成。完成橫幅會告訴您如何繼續：

```text
Resume with: /gsd-autonomous --from 6
```

---

## 以互動式討論模式執行

預設情況下，自主模式使用智慧討論（批量表格提案）自動回答討論問題。如果您希望自己回答設計問題，同時將規劃和執行保持在主上下文之外：

```bash
/gsd-autonomous --interactive
```

互動模式下：
- `/gsd-discuss-phase` 內聯執行並等待您的回答
- 規劃和執行作為後臺代理分發，以便您在討論下一階段時當前階段仍在構建
- 主上下文保持精簡——只有討論對話會累積

---

## 哪些安全門控仍然適用

自主模式不會繞過 GSD 的品質流水線。每個階段仍然會：

- 在執行前執行計劃檢查器
- 執行後讀取 `VERIFICATION.md` 並根據結果進行路由
- 當驗證狀態為 `human_needed` 或 `gaps_found` 時暫停並詢問您的處理意見
- 任何步驟失敗時停止並提供選項（修復並重試、跳過階段或停止）

與手動執行的唯一區別是：`passed` 狀態的驗證會自動推進——除非需要做出決策，否則不會在階段間提示您。

包合法性門控也保持啟用。如果計劃中包含針對可疑包的 `checkpoint:human-verify` 任務，執行器將停止並顯示檢查點。自主模式不會靜默安裝被標記的包。

---

## 何時不使用自主模式

以下情況請勿使用 `/gsd-autonomous`：

- **階段存在未解決的設計決策。** 如果您尚未執行 `/gsd-discuss-phase` 且 `PROJECT.md` 未捕獲您的偏好，智慧討論將做出您可能不認同的自主選擇。請先互動式地執行討論，或使用 `--interactive`。

- **您需要對單個階段進行精細控制。** 對於單個階段，`/gsd-execute-phase N` 會提供逐步輸出並允許您在繼續之前做出反應。自主模式專為批次無人值守執行而設計。

- **階段包含新穎或高風險的工作。** 自主模式除非遇到阻礙，否則會跳過暫停。對於預期有意外情況的階段，請通過手動執行保持在迴圈中。

- **您正處於已部分執行的階段中途。** 自主模式可以接管未完成的階段，但不會恢復部分執行的波次。使用 `/gsd-execute-phase N` 完成已在進行中的階段。

如果執行中途停止，請參閱[除錯失敗的執行](debug-a-failed-execution.md)瞭解如何診斷問題所在。

---

## 執行期間檢查進度

自主模式在每個階段前列印進度橫幅：

```text
 GSD ► AUTONOMOUS ▸ Phase 3/7: Auth Middleware [████░░░░] 28%
```

如果您需要在會話中途檢視執行進度，請開啟另一個終端並執行：

```bash
/gsd-progress
```

---

## 停止後恢復

如果自主模式停止——無論是您從阻礙提示中選擇了"停止自主模式"，還是會話被中斷——請從停止處繼續：

```bash
/gsd-autonomous --from 4     # 將 4 替換為第一個未完成的階段編號
```

GSD 會自動跳過已完成的階段，因此如果您不確定執行在何處停止，從較早的階段編號重新執行是安全的。
