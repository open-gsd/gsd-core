**`state.planned-phase` no longer silently no-ops on a non-canonical STATE.md** — when STATE.md carried no labeled `Status:` / `Total Plans in Phase:` field for the transition to advance (e.g. a fully narrative `## Current Position`), the transition matched nothing, returned `{"updated":[]}`, exited 0 and wrote zero bytes, yet `plan-phase` step 13b claimed success — so planning completed while the authoritative cross-tool state still said `ready_to_plan`. The transition now classifies its outcome (`applied` / `idempotent` / `unsupported-shape`); an unsupported shape fails loud with a structured error and a non-zero exit so `plan-phase` (and `plan-review-convergence`) stop and ask for reconciliation instead of continuing on stale state, and both step 13b and the convergence workflow inspect that result before claiming planning was recorded. `idempotent` is reported distinctly from `applied` and never implies the Status reached "Ready to execute" — an executor-authored Status (#397) is preserved across the no-op. The advertised `--name` argument — previously parsed and discarded by the router — now reaches the transition and repairs the Current Position phase name, interpolated literally so `$`-metacharacters and parenthesized suffixes are not corrupted. The frontmatter `progress.total_plans` is a milestone-wide metric distinct from the per-phase "Total Plans in Phase" body field, so planned-phase never writes the per-phase count into it (#500); it takes the derived milestone value under the per-counter ratchet (#1446/#2440 — `completed_*` stay monotonic, the totals always re-derive). (#2400)

**STATE.md transitions preserve the document's line ending** — every mutating state verb (`planned-phase`, `begin`/`complete phase`, `advance-plan`, `milestone switch`/`complete`, `sync`, `rebuild`, `patch`, `update`) re-attached frontmatter through per-caller template literals that hardcoded `\n`, so a CRLF STATE.md came back with an LF frontmatter block above a still-CRLF body. The nine call sites now share one `reassembleFrontmatter` seam in the frontmatter module that emits the block in the line ending the document was authored in, and the milestone section-reset paths compose their replacement bodies the same way. On-disk output was already normalized to LF by the `.md` write seam, so this closes the corruption for library/SDK consumers of `transitionCore` and removes the trap for any future EOL-preserving writer. The `planned-phase` shape probe additionally reads a fence-stripped copy of the body, so an illustrative `Status:` inside a fenced example can no longer certify a narrative STATE.md as advanceable — falling back to the raw body when the fence scan reports an unterminated fence, so an unclosed fence in a nested list can never mask a real `Status:` and hard-fail a well-formed STATE.md. (#2400)
